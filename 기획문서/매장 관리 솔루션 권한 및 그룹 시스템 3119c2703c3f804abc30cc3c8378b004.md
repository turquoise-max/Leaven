# 매장 관리 솔루션 권한 및 그룹 시스템

# [Spec] 매장 관리 솔루션 권한 및 그룹 시스템 (Discord-like RBAC)

## 1. 개요 (Overview)

본 시스템은 매장 내 인력의 **'접근 제어(Security)'**와 **'운영 효율(Operation)'**을 동시에 달성하기 위해 **Role(역할)**과 **Group(그룹)**을 분리하여 설계한다.

- **Role (역할):** 시스템 접근 권한(Permission)을 제어하는 **단일** 계급. (예: 점주, 매니저, 스태프)
- **Group (그룹):** 스케줄링, 필터링, 알림 타겟팅을 위한 **다중** 속성 태그. (예: 오전조, 주방팀, 초보자)

---

## 2. 데이터 모델링 (Database Schema)

### 2.1. ERD 구조 요약

- **StoreMember** (N) : **Role** (1) $\rightarrow$ *한 명은 하나의 역할만 가짐*
- **StoreMember** (N) : **Group** (N) $\rightarrow$ *한 명은 여러 그룹에 속함 (M:N)*

### 2.2. 테이블 상세 정의

### **A. Roles (역할 정의 테이블)**

각 매장별로 커스텀 역할을 생성할 수 있다.

JSON

`{
  "id": "role_uuid_1",
  "store_id": "store_A",
  "name": "매니저",
  "color_hex": "#3498DB", // 디스코드 스타일 색상
  "rank": 2, // 위계 (1: Owner ~ 99: Lowest). 높은 랭크가 낮은 랭크 관리 가능
  "permissions": [ // 활성화된 권한 키 목록
    "schedule.edit",
    "member.invite",
    "notice.create"
  ],
  "is_default": false, // 신규 가입 시 기본 부여 여부
  "is_uneditable": false // Owner 역할 삭제 방지
}`

### **B. Groups (그룹/태그 정의 테이블)**

권한과 무관한 운영상의 분류.

JSON

`{
  "id": "group_uuid_1",
  "store_id": "store_A",
  "name": "오픈조",
  "badge_icon": "☀️", // (Optional) 시각적 구분용 아이콘
  "description": "평일 07:00 ~ 13:00 근무자"
}`

### **C. StoreMember (매장 멤버 테이블 - 연결)**

JSON

`{
  "user_id": 101,
  "store_id": "store_A",
  "role_id": "role_uuid_1", // Foreign Key (Not Null)
  "group_ids": ["group_uuid_1", "group_uuid_2"] // Array or Junction Table
}`

## 3. 기능 요구사항 (Functional Requirements)

### 3.1. Role (역할) 관리 로직

1. **단일 할당 원칙:** 한 명의 직원은 반드시 **1개의 Role**만 가질 수 있다. (수정 시 기존 Role은 대체됨)
2. **초기 프리셋 (Default Presets):** 매장 생성 시 아래 3개 역할은 자동 생성된다.
    - 👑 **점주 (Owner):** Rank 1 / 모든 권한 ON / 수정·삭제 불가 / 양도만 가능.
    - 🧢 **매니저 (Manager):** Rank 2 / 급여 제외, 운영 권한 ON.
    - 🐥 **스태프 (Staff):** Rank 3 / 조회 권한 ON / `is_default: true`.
3. **권한 검증 (Guard):** API 요청 시 `User.Role.permissions` 배열에 해당 기능의 Key가 있는지 확인한다.

### 3.2. Group (그룹) 관리 로직

1. **다중 할당:** 0개 이상의 그룹을 가질 수 있다.
2. **권한 무관:** 그룹을 추가하거나 삭제해도 접근 권한(Permissions)에는 영향을 주지 않는다.
3. **활용처:**
    - **스케줄링:** "오픈조" 그룹만 필터링하여 스케줄 복사.
    - **알림(Push):** "주방팀" 그룹에게만 공지사항 푸시 발송.
    - **뱃지:** 직원 리스트나 채팅창 등에서 이름 옆에 뱃지 노출.

## 4. 권한 매트릭스 (Permission Keys)

개발 시 코드에서 사용할 권한 상수(Constant) 목록.

### **[Category: HR & Store]**

- `store.settings.edit`: 매장 정보(이름, 주소) 수정
- `member.manage`: 직원 초대, 승인, 추방, 정보 수정
- `role.manage`: 역할 생성 및 권한 수정 (본인 Rank보다 낮은 역할만 가능)

### **[Category: Operation]**

- `schedule.create`: 근무표 생성 및 수정
- `schedule.view_all`: 동료들의 스케줄 조회 (Off 시 본인 것만 보임)
- `attendance.correct`: 직원들의 출퇴근 기록 강제 수정 (지각/조퇴 처리)

### **[Category: Finance (Sensitive)]**

- `sales.view_daily`: 일별 매출 대시보드 접근
- `salary.view_all`: 전 직원 시급 및 급여 대장 조회 **(주의)**
- `salary.set`: 직원별 시급 설정

## 5. UI/UX 가이드 (Wireframe Guide)

### 5.1. 역할 및 권한 설정 페이지

- **Layout:** 2-Column (좌측: 역할 리스트 / 우측: 권한 토글)
- **Action:**
    - `[+ 역할 추가]` 버튼 클릭 시 팝업 $\rightarrow$ 이름/색상 입력.
    - 우측 패널에서 각 권한 항목별 `[ON/OFF]` 스위치 조작.
    - *"저장되지 않은 변경사항이 있습니다"* 하단 플로팅 바 노출 (디스코드 방식).

### 5.2. 직원 상세 프로필 (멤버 관리)

- **Role (Radio Button):**
    - 
    - 수직 나열된 리스트 중 **하나만 선택** 가능.
    - 선택 시 해당 역할의 색상이 프로필 테두리에 반영됨.
- **Group (Multi-select Chips):**
    - `[+ 태그 추가]` 버튼 클릭.
    - 드롭다운에서 여러 개 체크 가능.
    - 선택된 그룹은 알약 모양 칩(Chip)으로 나열됨 `(x)오픈조` `(x)마감조`.

### 5.3. 스케줄링 화면 (필터링 적용)

- **Filter Bar:** 상단에 `모든 직원 ▾` 드롭다운 배치.
- **Interaction:**
    - 드롭다운 클릭 시 생성된 **Group 목록** 노출.
    - `오픈조` 선택 시 $\rightarrow$ 해당 그룹 태그를 가진 직원 행(Row)만 남기고 나머지 숨김.

## 6. 예외 처리 및 정책 (Policy)

1. **Role 삭제 정책:**
    - 해당 Role을 할당받은 직원이 1명이라도 있다면 삭제할 수 없다. (안내 메시지: *"이 역할을 가진 직원이 있습니다. 먼저 역할을 변경해주세요."*)
2. **Owner 유일성:**
    - 한 매장에 Owner 권한을 가진 사람은 **오직 1명**이어야 한다.
3. **Role Rank 보호:**
    - 매니저는 자신보다 높은 Rank(점주)의 역할을 수정하거나 할당할 수 없다.
